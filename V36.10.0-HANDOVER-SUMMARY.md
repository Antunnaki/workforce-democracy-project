## üöÄ How to Deploy

### Quick Deploy (5 minutes)

**Frontend (Netlify):**
```bash
# Push to Git - Netlify auto-deploys
git add .
git commit -m "V36.10.0 Phase 1: Civic Representative Finder"
git push origin main
```

**Backend (VPS):**
```bash
# SSH to server
ssh workforce-backend

# Navigate to backend directory
cd /var/www/workforce-backend

# Backup current server.js
cp server.js server.js.backup-$(date +%Y%m%d-%H%M%S)

# Edit server.js and add the endpoint
# (See BACKEND-CIVIC-REPRESENTATIVES-API.md for code)
nano server.js

# Restart backend
/opt/nodejs/bin/pm2 restart workforce-backend

# Check logs
/opt/nodejs/bin/pm2 logs workforce-backend --lines 50
```

**Detailed Instructions:** See `DEPLOY-V36.10.0-PHASE1-INSTRUCTIONS.md`

## üí¨ Questions & Support

### Common Questions

**Q: Do I need to deploy frontend AND backend?**  
A: Yes! Frontend handles UI, backend handles API calls. Both needed for full functionality.

**Q: What if backend deployment fails?**  
A: Frontend will still work, but representative lookup will show errors. Use rollback instructions in deployment guide.

**Q: Can I test without deploying backend?**  
A: Frontend will render, but API calls will fail. You'll see error messages when trying to find representatives.

**Q: How do I update Congress.gov API key?**  
A: Edit `/var/www/workforce-backend/.env` on VPS, add `CONGRESS_GOV_API_KEY=your_key_here`

**Q: Why mock data in Phase 1?**  
A: To test the complete UI/UX flow before integrating real API. Phase 2 adds real data.

### Troubleshooting

**Issue: Representatives not displaying**  
‚Üí Check backend logs: `pm2 logs workforce-backend`  
‚Üí Verify endpoint added to server.js  
‚Üí Test endpoint manually with curl

**Issue: Encryption errors**  
‚Üí Verify security.js is loaded  
‚Üí Check browser console for errors  
‚Üí Clear localStorage and try again

**Issue: UI not loading**  
‚Üí Hard refresh: Ctrl+Shift+R  
‚Üí Check browser console for JS errors  
‚Üí Verify civic-representative-finder.js loaded
