## üìö Technical Explanation

### Why `width: 180px` Instead of `min-width`?

**`min-width: 160px` (Wrong):**
- Sets MINIMUM width
- Allows buttons to GROW if content needs more space
- Result: Different sizes for different text lengths

**`width: 180px` (Correct):**
- Sets EXACT width
- Buttons CANNOT grow beyond 180px
- Result: All buttons exactly the same size

### Why `flex: 0 0 auto` Required?

**Without it:**
```css
width: 180px;
/* But flex items can still grow/shrink by default! */
```

**With it:**
```css
flex: 0 0 auto; /* Don't grow (0) or shrink (0) */
width: 180px;   /* Use this exact width */
```

**Result:** Flex container respects the exact 180px width

### Why Not Just `flex: 0 0 180px`?

**Option 1 (Not Used):**
```css
flex: 0 0 180px; /* flex-basis: 180px */
```

**Option 2 (Used - Better):**
```css
flex: 0 0 auto;  /* flex-basis: auto */
width: 180px;    /* Use width property */
```

**Why Option 2 is Better:**
- More explicit (width property is clear)
- Easier to maintain (width is standard CSS)
- Better readability (obvious what controls sizing)
