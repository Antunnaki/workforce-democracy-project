## Root Cause Analysis

### Why did this happen?

**Header Issue:**
- `unified-color-scheme.css` was added for consistent styling
- It applied gradient text effect globally to `.site-title`
- This conflicted with white text needed for header
- Transparent text on light background = invisible

**Button Issue:**
- CSS variables `var(--text)` and `var(--border)` were never defined
- Should have been `var(--text-primary)` and `var(--border-light)`
- No hover/active color states defined
- Result: invisible or barely visible text

**Prevention:**
- Use specific CSS selectors (`.hero-section .site-title` vs `.site-title`)
- Always use defined CSS variables from color scheme
- Test on all pages after making global CSS changes
- Document variable naming conventions

## Lessons Learned

1. **Specificity Matters**
   - Global CSS rules can cause unintended side effects
   - Use specific selectors (`.hero-section .title` vs `.title`)

2. **Test Everywhere**
   - Changes to shared CSS files must be tested on ALL pages
   - What works on homepage might break on other pages

3. **Use Defined Variables**
   - Always use CSS variables from the unified color scheme
   - Check variable definitions before using them

4. **Visual Testing is Critical**
   - Code might work but be invisible due to color issues
   - Always visually inspect after CSS changes

5. **Document Conventions**
   - Clear documentation prevents future conflicts
   - Variable naming conventions should be documented

---

## Future Prevention Checklist

When making CSS changes:

- [ ] Check if change affects global elements (`.site-title`, `.logo`, etc.)
- [ ] Use specific selectors when needed (`.hero-section .title`)
- [ ] Verify CSS variables are defined in color scheme
- [ ] Test on all pages (index, learning, privacy, faq, philosophies, help)
- [ ] Test all interactive states (default, hover, active, focus)
- [ ] Check color contrast with WCAG tools
